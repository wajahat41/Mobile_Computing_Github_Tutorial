What is Github

Github is a version control systemt that can be used to to create better software through keeping track of all the changes over time. It also provides a way to collaborate with other developers on Projects.
It is used by developers all over the world and is one of the largest software hosting website, with millions of repositries.

You can visit GitHub This Tutorial will cover some basic ways in which github can be used.

Creating Account

You can visit GitHub website and create an account for free
Follow following steps to create an account

  1. Click on Sign up
  2. Provide basic details
  3. Verify Email

And you have created a GitHub account.

Creating Repository On Github

A GitHub repository is a place that contains all of your files and folders for a specific project. Ideally, Each project should have it's seperate repository.

  1. Click on New
  2. Provide Repository Name
  3. Provide Repository desrciption
  4. Choose visibility (Public or Private)
  5. Click on Create

And you have created your first repository.

Local Local Git Repository

A Local Git repository is a version control system that resides on the machine that you are working on.

Starting Local Repository

Starting the local repository is the first most step you will take. And it is fairly simple. You can use following command to start a local repository.
Make sure that you are in the directory where you want to start the repo.

$ git init

Staging Area

Once you make changes to a file, first step is moving those files to a staging area. You can use following command to move files to a staging area.
|filename| is the name of file. You can use . to move all files to staging area.

$git add | filename |

Status

Status shows if the files have been modified since last commit. If it if first time, status will show the files as unmarked. You can use following command to Check status of repository.

$git status

Committing

Commting is the stage in which you are actually saving the changes to repository. You can use following command to move files to commit changes

$git commit -m "This is where you put commit message"


Remote Respositories

In the previous step we created a local repository that is saved on your machine. Now, we are ready to connect this local respostory to a remote respostory, that will make our code availble on GitHub.

Connecting to remote respository

Connecting to a remote repository means creating a link between our local respository and a repository on github. We will need the link of github respository for this step. You can use following command to connect to a remote repository.
| link | is the link of remote repository.

$ git remote add origin | link |


Git pull

Once we have connected the local and remote repositories we can now copy the code from our remote repository to our local repository and start working on the project. You can use following command to copy files from remote to local repo.

$ git pull


Git push

After making changes to our local repository, we can commit these changes but this commit will take effect only in local repository. To save these changes to remote repository we will have to push the commits. You can use following command to save changes to remote repo.

$ git push
